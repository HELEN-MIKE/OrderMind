# OrderMind 客户试用演示说明

日期: 2026-07-04

## 演示目标

用内置脱敏仿真订单跑通客户最关心的闭环:

1. 打开桌面应用或本地 Web 应用。
2. 使用脱敏示例订单完成审单。
3. 查看正常订单的明细抽取和零问题结果。
4. 查看问题订单的错误、警告和定位信息。
5. 导出 HTML 报告和 Excel 报告。

所有示例订单都位于 `samples/customer_like_orders/`，公司名、联系人、地址、电话和订单号均为虚构数据。

## 推荐演示顺序

### 1. 正常中文采购订单

首页点击 `中文采购订单`。

预期结果:

- 来源文件为 `domestic_purchase_order_zh.txt`。
- 明细行数为 3。
- 错误数为 0，警告数为 0。
- 明细表展示货号、品名、数量、单价、小计、材质。

这一步用于证明中文订单的基础解析、金额汇总和规则校验可用。

### 2. 英文商业发票

返回首页，点击 `英文商业发票`。

预期结果:

- 来源文件为 `commercial_invoice_en.csv`。
- 明细行数为 3。
- 错误数为 0，警告数为 0。
- 付款方式、包装要求、交货期可在结构化 JSON 中看到。

这一步用于证明英文订单字段别名可以被识别，例如 `Item No`、`Description`、`Qty`、`Amount`。

### 3. Excel 多币种订单

返回首页，点击 `Excel 多币种订单`。

预期结果:

- 来源文件为 `multi_currency_order.xlsx`。
- 明细行数为 3。
- 错误数为 0，警告数为 0。
- 订单金额按核心 `Amount` 列校验，多币种备注作为订单附加列保留在原始证据中。

这一步用于证明桌面试用包可以在离线环境解析 `.xlsx` 文件。

### 4. 文本型 PDF 订单

返回首页，点击 `文本型 PDF 订单`。

预期结果:

- 来源文件为 `text_pdf_order.pdf`。
- 明细行数为 2。
- 错误数为 0，警告数为 0。
- 付款方式、包装要求、交货期和订单明细可被提取。

这一步用于证明可复制文字的电子 PDF 已经可以进入同一套审单流程。扫描件 PDF 会在文本抽取失败后尝试调用本机 OCR 命令。

### OCR 扫描件试用

当前版本支持把扫描件 PDF 和图片订单接入同一套审单流程，但 OCR 引擎需要客户电脑本机提供。

默认命令:

```bash
tesseract 文件 stdout -l chi_sim+eng
```

也可以通过环境变量指定自定义命令:

```bash
ORDERMIND_OCR_COMMAND=/path/to/ocr-command
```

自定义命令只需要接受文件路径参数，并把识别文本输出到标准输出。若本机没有可用 OCR 命令，OrderMind 会提示 OCR 命令不可用，而不会把它误判成订单内容错误。

### 5. 故意带问题样例

返回首页，点击 `故意带问题样例`。

预期结果:

- 来源文件为 `review_findings_bad_amount_missing_material.txt`。
- 明细行数为 3。
- 错误数为 3，警告数为 3。
- 校验问题包含:
  - `必填字段缺失: product_name`
  - `小计金额不一致`
  - `总金额不一致`
  - `单价小数位数超过 2 位`
  - `货号格式不符合规则`
  - `单位不在允许范围内`

这一步用于演示 OrderMind 不是只做字段抽取，而是能指出需要人工复核的具体问题。

## 报告导出验收

在任意结果页点击:

- `导出报告`: 下载独立 HTML 报告。
- `导出 Excel`: 下载 `.xlsx` 报告。

预期结果:

- HTML 报告可在浏览器离线打开。
- Excel 报告包含 `摘要`、`订单明细`、`校验问题`、`结构化JSON` 四个工作表。
- 文件名会使用来源订单名生成，例如 `commercial_invoice_en-report.xlsx`。

## 演示边界

- 当前版本支持 TXT、CSV、TSV、XLSX、XLSM、PDF 和常见图片格式。
- PDF 会优先抽取可复制文本，扫描件 PDF 和图片会回退到本机 OCR 命令。
- OCR 只负责把图像转成文本；识别质量、表格边界和错行仍需人工复核。
- 复杂合并单元格、跨页表格和语义级条款理解仍需后续文档智能能力扩展。
